14-http-describedby-citeas-linkset-json-txt-conneg

This landing page has four HTTP headers:

  • describedby to index.ttl (text/turtle, type attribute declared)
  • cite-as to w3id
  • linkset to linkset with content negotation for application/linkset+json that links to test-apple-data.csv and the above
  • linkset to linkset with content negotation for application/linkset alternative to the above

When requesting the linkset resource The Web Server should give a specific Content-Location header of which resource was resolved, in addition to rel=alternate link between JSON and TXT, as they have the same public URI as well as a rel=canonical for the untyped URI - however neither of these are considered part of the signposting.

Bonus: Content types application/json and text/plain also content-negotiate with lower priority.
