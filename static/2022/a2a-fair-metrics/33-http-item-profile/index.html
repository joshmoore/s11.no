33-http-item-profile

This landing page has one HTTP Link headers:

  • item to crate-33.zip (type application/zip with profile https://w3id.org/ro/crate to indicate the ZIP archive is an RO-Crate)

The media type application/zip have not got a profile attribute defined, therefore the Content-Type of the zip file is unchanged, however a rel=profile Link header is included when requesting the zip, corresponding to the profile attribute indicated on the rel=item link here.

The RO-Crate contains further metadata and PID embeded as JSON-LD, but in this case that is not exposed by the web server as we can't describe items inside the ZIP file here.
