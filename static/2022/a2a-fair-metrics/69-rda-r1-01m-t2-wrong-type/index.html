69-rda-r1-01m-t2-wrong-type

This landing page has these FAIR Signposting HTTP headers:

  • rel=cite-as to w3id PID
  • rel=item to CSV file (wrong type=csv)

This tests RDA-R1-01M, test 2: type attribute follows the https://www.rfc-editor.org/rfc/rfc9110.html#section-8.3 format of type/subtype with optional ;properties. This is a negative test because csv does not follow the type/subtype pattern.
