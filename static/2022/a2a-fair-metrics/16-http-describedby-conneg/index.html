16-http-describedby-conneg

This landing page has two signposting HTTP headers:

  • describedby content-negotiated to metadata.ttl (text/turtle, type attribute declared)
  • describedby content-negotiated to metadata.jsonld (application/ld+json, type attribute declared)

The client is forced to use HTTP content negotiation in this scenario as both links have metadata as target. The server will add a Content-Location of the seleted resource.
